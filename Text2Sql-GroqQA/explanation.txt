WHAT IS SQLITE?
--------------------------------------------------------------------------------
SQLite is a lightweight, serverless, self-contained SQL database engine.

Key Features:
- Zero Configuration: No setup or administration needed
- Serverless: No separate server process required
- Single File: Entire database stored in one file (.db)
- Cross-Platform: Works on Windows, Mac, Linux, mobile
- ACID Compliant: Ensures data integrity

Why I Used SQLite:
- Perfect for small to medium applications
- No installation required - comes with Python
- Great for prototyping and learning
- Easy to deploy and share

Real-World Use Cases:
- Mobile apps (Android, iOS)
- Embedded systems
- Desktop applications
- Testing and development
- Small websites

FILE EXPLANATIONS
--------------------------------------------------------------------------------

1. app.py (Main Application)
-----------------------------
   This is the main file that runs the Streamlit web application.

   Key Components:

   a) Imports and Configuration:
      - Loads environment variables from .env file
      - Imports Streamlit for web UI
      - Imports Groq client for AI model access
      - Configures Groq client with API key

   b) get_groq_response(question, prompt):
      - Takes user's question and system prompt as input
      - Sends request to Groq API using LLaMA 3.3 70B model
      - Returns generated SQL query as response
      - Uses chat completion API with system and user messages

   c) read_sql_query(sql, db):
      - Connects to SQLite database
      - Executes the SQL query generated by the AI
      - Returns fetched rows from database
      - Handles database connection and cursor management

   d) Prompt:
      - Contains instructions for the AI model
      - Explains database structure (STUDENT table)
      - Provides examples of question-to-SQL conversion
      - Instructs model to not include ``` or "sql" word in output

   e) Streamlit UI:
      - Creates text input for user questions
      - Submit button to trigger query generation
      - Displays results using st.header() for each row


2. sqlite.py (Database Setup)
-----------------------------
   This script creates and populates the SQLite database.

   Steps:
   a) Connects to SQLite database (creates student.db if not exists)
   b) Creates STUDENT table with columns:
      - NAME (VARCHAR 25)
      - CLASS (VARCHAR 25)
      - SECTION (VARCHAR 25)
      - MARKS (INT)
   c) Inserts sample records:
      - Pramjeet - Data Science - A - 90
      - Vinay - Data Science - B - 100
      - Adarsh - Data Science - A - 86
      - Sumit - DEVOPS - A - 50
      - Shyam - DEVOPS - A - 35
   d) Commits changes and closes connection

   Note: Run this file only ONCE to set up the database.


3. requirements.txt (Dependencies)
----------------------------------
   Lists all Python packages needed:
   - streamlit      : Web application framework
   - groq           : Groq API client for LLaMA models
   - python-dotenv  : Load environment variables from .env


4. .env (Environment Variables)
-------------------------------
   Contains sensitive configuration:
   - GROQ_API_KEY=your_groq_api_key_here

   Get your API key from: https://console.groq.com/keys


HOW THE APP WORKS (FLOW)
--------------------------------------------------------------------------------
User asks: "Show students with marks above 80"
     |
     v
Groq API (LLaMA model) converts to SQL:
     "SELECT * FROM STUDENT WHERE MARKS > 80"
     |
     v
SQL executes on SQLite database
     |
     v
Results displayed to user


HOW TO RUN THE PROJECT
--------------------------------------------------------------------------------
Step 1: Install dependencies
   pip install -r requirements.txt

Step 2: Set up environment variables
   Create .env file with your Groq API key:
   GROQ_API_KEY=your_key_here

Step 3: Initialize database (only once)
   python sqlite.py

Step 4: Run the application
   streamlit run app.py

INTERVIEW QUESTIONS I CAN ANSWER
--------------------------------------------------------------------------------
Q: Why Groq instead of OpenAI?
A: Groq offers faster inference speeds and cost-effective pricing
   for LLaMA models, making it ideal for real-time applications.

Q: Why SQLite over MySQL/PostgreSQL?
A: SQLite is serverless and perfect for prototyping. For production
   with multiple users, I would migrate to PostgreSQL.

Q: How do you prevent SQL injection?
A: The AI generates read-only SELECT queries. For production, I would
   add query validation and use parameterized queries.

Q: What if the AI generates wrong SQL?
A: I handle exceptions and show user-friendly error messages.
   The prompt includes examples to improve accuracy.

Q: How would you scale this application?
A: Use a production database (PostgreSQL), add caching,
   implement rate limiting, and deploy on cloud (AWS/GCP).
